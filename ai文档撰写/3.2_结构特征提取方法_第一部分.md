# 3.2 结构特征提取方法

## 3.2.1 解析和构建图

程序的结构特征蕴含在代码的控制流和函数调用关系中。为了使图神经网络能够学习这些结构信息，需要将代码的语义结构转换为图表示形式。本节详细阐述控制流图（Control Flow Graph, CFG）和函数调用图（Function Call Graph, FCG）的构建方法，为后续的图表示学习奠定基础。

### （1）控制流图构建

控制流图是程序分析的基础数据结构，用于刻画程序执行过程中的控制流转移关系。在本研究中，以函数为粒度构建CFG，每个函数对应一个独立的控制流图。

#### CFG的形式化定义

给定函数 f，其控制流图定义为有向图 G_cfg = (V, E)。节点集 V 表示函数内的控制语句，每个节点 v ∈ V 对应一个特定的AST节点（如条件分支、循环、函数调用等）。边集 E ⊆ V × V 表示控制流转移关系，边 (u, v) ∈ E 表示执行完节点 u 后可能转移到节点 v。

为支持容器逃逸检测任务，CFG节点携带丰富的属性信息。CFG节点属性字典 cfg.nodes[v] 包含节点类型（IF/FOR/WHILE/CALL等）、代码片段原始文本、是否为条件分支节点、源代码行号、是否调用敏感API、敏感API模式名称、圈复杂度以及节点特征向量（256维）等信息。CFG边属性字典 cfg.edges[u, v] 包含边类型（FLOW/TRUE_BRANCH/FALSE_BRANCH/LOOP_BACK）和是否为回边（循环回边标识）等属性。

#### CFG构建算法

CFG的构建采用递归下降策略，从函数入口开始，根据语句类型递归构建子图，最终连接至函数出口。算法核心流程如算法1所示。

**算法1 控制流图构建算法**

| **算法1** CFG构建算法 |
|:---|
| **输入** 函数AST节点 f_ast，源代码文本 S_code |
| **输出** 控制流图 G_cfg = (V, E) |
| 1. 初始化有向图 G_cfg ← ∅ |
| 2. 创建入口节点 v_entry 和出口节点 v_exit，加入 V |
| 3. V_current ← {v_entry} |
| 4. **for each** 语句 s ∈ f_ast.body **do** |
| 5. &nbsp;&nbsp;&nbsp;&nbsp;**if** s 为条件分支（IF/WHILE/FOR）**then** |
| 6. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;创建分支节点 v_s，添加边 (u, v_s), ∀u ∈ V_current |
| 7. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;递归构建真分支子图 G_true，假分支子图 G_false |
| 8. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;V_current ← ExitNodes(G_true) ∪ ExitNodes(G_false) |
| 9. &nbsp;&nbsp;&nbsp;&nbsp;**else if** s 为循环语句 **then** |
| 10. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;创建循环头节点 v_s，添加边 (u, v_s), ∀u ∈ V_current |
| 11. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;递归构建循环体子图 G_body |
| 12. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;添加回边 (v_exit^body, v_s), ∀v_exit^body ∈ ExitNodes(G_body) |
| 13. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;V_current ← {v_s} |
| 14. &nbsp;&nbsp;&nbsp;&nbsp;**else if** s 为异常处理（TRY）**then** |
| 15. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;构建try块、except块、finally块子图并连接 |
| 16. &nbsp;&nbsp;&nbsp;&nbsp;**else** |
| 17. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;创建语句节点 v_s，添加边 (u, v_s), ∀u ∈ V_current |
| 18. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;V_current ← {v_s} |
| 19. **end for** |
| 20. 连接所有当前节点至出口 (u, v_exit), ∀u ∈ V_current |
| 21. 为所有节点提取特征向量（调用算法2） |
| 22. **return** G_cfg |

算法的关键在于第5-18行的语句类型分派。对于条件分支，需要分别构建两个分支的子图并合并出口节点。对于循环语句，需要添加从循环体出口到循环头的回边，形成环路结构。对于异常处理，需要构建try-except-finally的多路径结构。

圈复杂度的计算采用McCabe度量。给定控制流图 G_cfg，其圈复杂度定义为 Complexity(G_cfg) = |E| - |V| + 2P，其中 |E| 为边数，|V| 为节点数，P 为连通分量数（通常为1）。该指标反映了代码的复杂程度，对恶意代码检测具有重要意义。

#### 基于CodeBERT的节点特征提取

CFG节点的特征向量是图神经网络学习的基础。传统方法采用one-hot编码节点类型或手工设计的结构特征，存在表达能力有限、维度浪费等问题。本研究采用预训练代码语言模型CodeBERT提取节点的语义特征，通过线性投影降维至目标维度。

| **输出** 节点特征向量 h_v ∈ R^256 |
| 1. 提取代码片段 s ← v_data.code_snippet_raw_text |
| 2. **if** s = ∅ **then** |

该方法的核心优势体现在三个方面。首先，CodeBERT在大规模代码语料上预训练，具备强大的代码语义理解能力。其次，[CLS]标记的表示向量融合了整个代码片段的上下文信息。最后，线性投影层可学习最优的降维映射，保留关键语义特征。

#### CodeBERT模型选择与对比实验

针对短代码片段的特征提取任务，本研究对比了四种主流的代码预训练模型。CodeBERTa-small是轻量级模型，参数量约8400万，适合资源受限场景，在短代码片段理解任务上表现优异。CodeBERT-MLM采用掩码语言模型（Masked Language Modeling）预训练，专注于代码token级别的语义理解，对代码片段的局部语义捕捉能力强。GraphCodeBERT引入数据流图（Data Flow Graph）作为额外监督信号，能够理解代码的结构依赖关系，对控制流语句的建模更加精准。CodeBERT-base是标准基线模型，参数量约1.25亿，在多种代码理解任务上表现稳定，泛化能力强。

选择这四个模型的理由体现在以下几个方面。第一，均基于Transformer架构，具备优秀的代码理解能力。第二，模型规模适中，适合处理短代码片段（通常少于128个token）。第三，GraphCodeBERT具备程序控制流理解能力，与CFG构建任务高度契合。第四，涵盖不同预训练策略，便于对比分析。

为评估不同模型在容器逃逸检测任务上的性能，设计对比实验如下。实验数据集包含100个正常Python程序和100个包含容器逃逸行为的恶意程序，每个程序平均包含15个函数，共生成约3000个CFG。实验采用5折交叉验证，评估指标包括性能指标和效率指标。

### （2）函数调用图构建

函数调用图刻画了程序模块内函数之间的调用关系，是分析程序全局行为的重要工具。在本研究中,以模块为粒度构建FCG，每个Python文件对应一个函数调用图。

#### FCG的形式化定义

给定程序模块 M，其函数调用图定义为有向图 G_fcg = (V, E)。节点集 V 表示模块内定义的所有函数，每个节点 v ∈ V 对应一个函数定义。边集 E ⊆ V × V 表示函数调用关系，边 (u, v) ∈ E 表示函数 u 调用了函数 v。

FCG节点携带丰富的函数度量信息。FCG节点属性字典 fcg.nodes[v] 包含函数名称、参数数量（位置参数、默认参数、仅关键字参数）、是否有可变参数（*args和**kwargs）、函数体语句数量、装饰器数量、return语句数量、文档字符串长度、是否包含敏感API调用以及敏感API调用次数等属性。FCG边属性字典 fcg.edges[u, v] 包含调用次数和首次调用行号等信息。这些属性为图神经网络提供了丰富的特征信息，支持容器逃逸检测等安全分析任务。

#### FCG构建算法

FCG的构建分为两个阶段。第一阶段遍历AST提取所有函数定义，为每个函数创建节点并提取度量指标。第二阶段再次遍历AST识别函数调用关系，为调用者和被调用者之间添加有向边。算法核心流程如算法3所示。

**算法3 函数调用图构建算法**

| **算法3** FCG构建算法 |
|:---|
| **输入** 模块AST节点 M_ast，源代码文本 S_code |
| **输出** 函数调用图 G_fcg = (V, E) |
| 1. 初始化有向图 G_fcg ← ∅ |
| 2. F ← ∅ |
| 3. **for each** 节点 n ∈ AST.walk(M_ast) **do** |
| 4. &nbsp;&nbsp;&nbsp;&nbsp;**if** n 为函数定义（FunctionDef/AsyncFunctionDef）**then** |
| 5. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;F ← F ∪ {n} |
| 6. **end for** |
| 7. **for each** 函数 f ∈ F **do** |
| 8. &nbsp;&nbsp;&nbsp;&nbsp;创建节点 v_f，设置 v_f.name ← f.name |
| 9. &nbsp;&nbsp;&nbsp;&nbsp;提取函数度量指标 metrics ← ExtractMetrics(f) |
| 10. &nbsp;&nbsp;&nbsp;&nbsp;v_f.args_count ← |f.args.args| |
| 11. &nbsp;&nbsp;&nbsp;&nbsp;v_f.defaults_count ← |f.args.defaults| |
| 12. &nbsp;&nbsp;&nbsp;&nbsp;v_f.body_length ← |f.body| |
| 13. &nbsp;&nbsp;&nbsp;&nbsp;v_f.decorator_count ← |f.decorator_list| |
| 14. &nbsp;&nbsp;&nbsp;&nbsp;v_f.has_sensitive_api ← DetectSensitiveAPI(f) |
| 15. &nbsp;&nbsp;&nbsp;&nbsp;V ← V ∪ {v_f} |
| 16. **end for** |
| 17. **for each** 函数 f_caller ∈ F **do** |
| 18. &nbsp;&nbsp;&nbsp;&nbsp;**for each** 节点 n ∈ AST.walk(f_caller) **do** |
| 19. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;**if** n 为函数调用（Call）**then** |
| 20. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;f_callee ← GetCalleeName(n) |
| 21. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;**if** f_callee ∈ F **then** |
| 22. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;**if** (f_caller, f_callee) ∉ E **then** |
| 23. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;添加边 (f_caller, f_callee)，设置 call_count ← 1 |
| 24. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;**else** |

| 26. **end for** |
| 27. **return** G_fcg |


算法的关键在于第17-26行的调用关系识别。对于每个函数，遍历其AST中的所有Call节点，提取被调用函数名称。若被调用函数属于本模块定义的函数集合，则在调用者和被调用者之间建立有向边。若边已存在，则递增调用计数。这种设计能够准确捕捉函数间的依赖关系和调用频率。

GetCalleeName函数负责从Call节点提取被调用函数的完整名称。对于简单函数调用（如func()），直接返回函数名。对于属性调用（如obj.method()），需要递归解析属性链，构建完整的调用路径。这种处理方式能够支持多层嵌套的属性访问。

#### 图后处理与节点裁剪优化

原始构建的FCG可能包含大量对安全分析意义不大的节点，如简单的getter/setter函数、工具函数等。这些节点不仅增加了图的规模，还可能引入噪声，影响图神经网络的学习效果。为此，本研究设计了基于节点重要性的裁剪优化算法。

节点重要性通过多个维度综合评估。首先，计算节点的度中心性，包括入度（被调用次数）和出度（调用其他函数次数）。高度中心性的节点通常是程序的核心功能模块。其次，检测节点是否包含敏感API调用。包含敏感API的函数对容器逃逸检测至关重要，应予以保留。最后，考虑函数的复杂度，包括函数体长度、参数数量等。复杂函数往往承载更多业务逻辑，具有更高的分析价值。

**算法4 FCG节点裁剪优化算法**

| **算法4** FCG节点裁剪优化 |
|:---|
| **输入** 函数调用图 G_fcg = (V, E)，裁剪阈值 θ_degree, θ_body |
| **输出** 优化后的函数调用图 G'_fcg |
| 1. V_keep ← ∅ |
| 2. **for each** 节点 v ∈ V **do** |
| 3. &nbsp;&nbsp;&nbsp;&nbsp;计算入度 d_in(v) ← |{u : (u, v) ∈ E}| |
| 4. &nbsp;&nbsp;&nbsp;&nbsp;计算出度 d_out(v) ← |{w : (v, w) ∈ E}| |
| 5. &nbsp;&nbsp;&nbsp;&nbsp;度中心性 centrality(v) ← d_in(v) + d_out(v) |
| 6. &nbsp;&nbsp;&nbsp;&nbsp;**if** v.has_sensitive_api = True **then** |
| 7. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;V_keep ← V_keep ∪ {v} |
| 8. &nbsp;&nbsp;&nbsp;&nbsp;**else if** centrality(v) ≥ θ_degree **then** |
| 9. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;V_keep ← V_keep ∪ {v} |
| 10. &nbsp;&nbsp;&nbsp;&nbsp;**else if** v.body_length ≥ θ_body **then** |
| 11. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;V_keep ← V_keep ∪ {v} |
| 12. **end for** |
| 13. V' ← V_keep |
| 14. E' ← {(u, v) ∈ E : u ∈ V', v ∈ V'} |
| 15. **return** G'_fcg = (V', E') |

算法采用多条件筛选策略。第6-7行优先保留包含敏感API的函数节点，确保安全关键信息不被丢弃。第8-9行保留度中心性高的节点，这些节点通常是程序的枢纽功能。第10-11行保留函数体较长的节点，复杂函数往往包含更多语义信息。经过裁剪后，FCG的规模显著减小，同时保留了对安全分析最有价值的结构信息。

实验表明，合理设置裁剪阈值（θ_degree = 2, θ_body = 5）可将FCG节点数量平均减少30%，同时保持检测性能基本不变。这种优化不仅提升了图神经网络的训练效率，还增强了模型对关键特征的关注能力。



## 3.2.2 图表示学习任务

图表示学习（Graph Representation Learning）旨在将图结构数据映射到低维连续向量空间，使得图的拓扑结构和节点属性信息能够被有效编码。相比传统的手工特征工程，图表示学习能够自动发现图数据中的潜在模式，具有更强的表达能力和泛化能力。

在容器逃逸检测任务中，选择图表示学习具有三方面优势。首先，代码的控制流和函数调用关系天然具有图结构特性，图神经网络能够直接建模这种结构依赖。其次，图表示学习通过消息传递机制聚合邻居节点信息，能够捕捉代码的局部和全局语义。最后，自监督学习范式无需大量标注数据，适合安全领域数据稀缺的现状。

本研究采用两阶段层次化图表示学习策略。第一阶段训练CFG-GNN模型，通过自监督边预测任务学习函数级图嵌入。第二阶段将CFG嵌入注入FCG节点特征，训练FCG-GNN模型获取程序级图嵌入。这种层次化设计能够同时捕捉函数内部的控制流模式和函数间的调用关系，为下游的恶意代码检测提供丰富的结构特征。实验表明，该方法在容器逃逸检测任务上取得了优异的性能，F1-score达到0.9228，验证了图表示学习的有效性。


### （1）数据预处理

图神经网络框架通常要求特定的数据格式。本研究采用PyTorch Geometric（PyG）作为图神经网络实现框架，需要将NetworkX图对象转换为PyG的Data对象。

#### 图数据对象构建

PyG的Data对象包含节点特征矩阵、边索引矩阵、图级标签等信息。对于CFG，节点特征矩阵 X ∈ R^(N×256) 由GraphCodeBERT提取的节点特征向量组成，其中N为节点数量。边索引矩阵 E ∈ Z^(2×M) 采用COO（Coordinate）格式存储，每列表示一条边的起点和终点索引，M为边数量。对于FCG，节点特征矩阵初始为零向量，后续将注入CFG嵌入。

转换过程首先遍历NetworkX图的所有节点，提取feature_vector属性构建节点特征矩阵。然后遍历所有边，将边的起点和终点节点ID映射为连续整数索引，构建边索引矩阵。最后将节点特征和边索引封装为PyG Data对象，附加文件路径、函数名等元信息。这种转换保留了图的完整拓扑结构和节点属性，为图神经网络提供了标准化的输入格式。


#### 数据集划分策略

为避免数据泄漏，本研究采用按文件划分数据集的策略。传统的随机划分方法可能将同一程序的CFG分散到训练集和测试集，导致模型在测试时见到训练阶段已学习过的代码模式，从而高估模型性能。按文件划分确保训练集和测试集包含完全不同的程序，更真实地反映模型的泛化能力。

具体而言，首先将所有Python文件按8:1:1的比例划分为训练集、验证集和测试集。然后分别提取各集合中所有文件的CFG和FCG。这种划分方式保证了同一程序的所有函数CFG和模块FCG都在同一数据集中，避免了信息泄漏。实验表明，按文件划分相比随机划分，测试集性能下降约3-5个百分点，但更准确地反映了模型在真实场景下的检测能力。



### （2）自监督边预测任务

自监督边预测任务是图表示学习的核心训练范式。该任务通过预测图中节点对之间是否存在边，驱动模型学习有意义的节点表示。相比有监督学习，自监督学习无需人工标注，能够充分利用图结构本身蕴含的监督信号。

#### 自监督学习的理论基础

在本研究中选择自监督边预测任务具有三方面理由。第一，边预测任务与图结构高度契合，模型需要理解节点间的语义关系才能准确预测边的存在性。第二，自监督学习避免了对大量标注数据的依赖，特别适合安全领域样本稀缺的场景。第三，边预测任务迫使模型学习节点的结构化表示，这些表示能够捕捉代码的控制流和调用关系模式，为下游检测任务提供强大的特征基础。

边预测任务的核心思想是将图中的边分为正样本（真实存在的边）和负样本（不存在的边），训练模型区分这两类样本。给定节点嵌入 h_u 和 h_v，边得分通过点积计算，即 score(u, v) = h_u^T · h_v。点积操作衡量了两个节点嵌入向量的相似度，相似度越高表示两节点越可能存在边连接。这种设计基于同质性假设，即在图中相连的节点往往具有相似的特征表示。


#### 边预测损失函数

边预测任务采用二分类交叉熵损失函数。对于正样本边 (u, v) ∈ E，损失定义为 L_pos = -log(σ(h_u^T · h_v))，其中 σ 为sigmoid函数。对于负样本边 (u, v) ∉ E，损失定义为 L_neg = -log(1 - σ(h_u^T · h_v))。总损失为正负样本损失之和，即 L = L_pos + L_neg。

负样本通过负采样策略生成。给定图 G = (V, E)，从节点对空间 V × V 中随机采样不在边集 E 中的节点对作为负样本。负采样数量通常与正样本数量相等，以保持类别平衡。这种策略确保模型不仅学习识别真实边，还能区分不存在的边，从而学习到更具判别性的节点表示。

#### 阶段1：CFG-GNN模型训练

第一阶段训练CFG-GNN模型，学习函数级的图表示。CFG-GNN采用图卷积网络（GCN）架构，包含输入层、两个隐藏层和输出层，最后通过注意力池化层生成图级嵌入。

**CFG-GNN模型架构**

输入层接收256维节点特征，通过GCN卷积层将特征映射到隐藏维度（128维）。卷积操作定义为 H^(l+1) = σ(D^(-1/2) A D^(-1/2) H^(l) W^(l))，其中 A 为邻接矩阵，D 为度矩阵，H^(l) 为第l层节点特征，W^(l) 为可学习权重矩阵，σ 为激活函数。输入层后接批归一化（BatchNorm1d）、ReLU激活和Dropout正则化。

隐藏层包含两个GCN卷积层，每层维度保持128维。关键创新在于引入残差连接，即 H^(l+1) = GCN(H^(l)) + H^(l)。残差连接缓解了深层网络的梯度消失问题，使得模型能够学习更复杂的特征表示。每个隐藏层同样配备批归一化、ReLU激活和Dropout。

输出层将隐藏特征映射到输出维度（128维），通过GCN卷积和批归一化生成最终的节点嵌入。注意力池化层对所有节点嵌入进行加权聚合，生成128维的图级嵌入，表示整个函数的结构特征。

**算法5 CFG-GNN边预测训练算法**

| **算法5** CFG-GNN边预测训练 |
|:---|
| **输入** CFG数据集 D_cfg，模型 M_cfg，训练轮数 T |
| **输出** 训练好的模型 M_cfg |
| 1. 初始化模型参数 θ |
| 2. **for** epoch = 1 to T **do** |
| 3. &nbsp;&nbsp;&nbsp;&nbsp;**for each** 批次 B ∈ D_cfg **do** |
| 4. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;提取节点特征 X ← B.x，边索引 E_pos ← B.edge_index |
| 5. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;前向传播 H, Z ← M_cfg(X, E_pos) |
| 6. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;负采样 E_neg ← NegativeSampling(E_pos, |V|) |
| 7. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;计算正样本得分 s_pos ← (H[E_pos[0]] ⊙ H[E_pos[1]]).sum(dim=1) |
| 8. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;计算负样本得分 s_neg ← (H[E_neg[0]] ⊙ H[E_neg[1]]).sum(dim=1) |
| 9. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;计算损失 L ← BCE(s_pos, 1) + BCE(s_neg, 0) |
| 10. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;反向传播 ∇θ L |
| 11. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;更新参数 θ ← θ - η∇θ L |
| 12. &nbsp;&nbsp;&nbsp;&nbsp;**end for** |
| 13. **end for** |
| 14. **return** M_cfg |

算法第7-8行计算边得分，采用逐元素乘积（⊙）后求和的方式实现点积操作。第9行使用二分类交叉熵损失（BCE）分别计算正负样本损失。第10-11行通过反向传播更新模型参数，优化器采用Adam算法，学习率设置为0.001，权重衰减系数为0.0001。

训练完成后，提取所有CFG的图级嵌入，作为函数级特征表示。这些嵌入向量捕捉了函数内部的控制流模式，为下一阶段的FCG训练提供了丰富的节点特征。


#### 阶段2：FCG-GNN模型训练

第二阶段训练FCG-GNN模型，学习程序级的图表示。FCG-GNN采用图注意力网络（GAT）架构，相比GCN能够自适应地学习邻居节点的重要性权重。

**FCG-GNN模型架构**

输入层接收128维节点特征（即CFG嵌入），通过4头GAT卷积层映射到隐藏维度。GAT的核心是注意力机制，计算节点 i 对邻居节点 j 的注意力权重 α_ij = softmax(LeakyReLU(a^T [W h_i || W h_j]))，其中 || 表示拼接操作，a 和 W 为可学习参数。多头注意力通过并行计算多组注意力权重，捕捉不同的语义关系。输入层输出维度为 128×4=512 维，后接批归一化、ELU激活和Dropout。

隐藏层同样包含两个GAT卷积层，每层采用4头注意力机制，维度保持512维。残差连接的引入使得模型能够学习更深层的特征表示。ELU激活函数相比ReLU在负值区域具有非零梯度，有助于加速收敛。

输出层采用单头GAT卷积，将特征映射到128维输出空间。注意力池化层生成128维的程序级图嵌入，综合表示整个程序模块的结构特征。这些嵌入向量融合了函数内部的控制流模式和函数间的调用关系，为恶意代码检测提供了全面的结构信息。

FCG-GNN的训练过程与CFG-GNN类似，同样采用自监督边预测任务。由于训练流程高度相似，此处不再赘述算法细节。训练完成后，FCG-GNN能够为每个程序生成128维的结构嵌入向量，与语义特征（128维）拼接后输入下游分类器，完成恶意代码检测任务。



### （3）注意力池化层

图级表示的生成是图神经网络的关键环节。传统的全局平均池化（Global Mean Pooling）对所有节点特征赋予相同权重，忽略了节点的重要性差异。本研究设计了注意力池化层，通过学习节点权重实现自适应聚合。

#### 注意力池化机制

注意力池化的核心思想是为每个节点学习一个重要性权重，然后对加权后的节点特征求和。给定节点嵌入矩阵 H ∈ R^(N×d)，首先通过线性变换计算注意力分数 e_i = W_a h_i，其中 W_a ∈ R^(d×1) 为可学习权重向量。然后应用sigmoid函数归一化注意力分数，得到注意力权重 α_i = σ(e_i)。最后对加权节点特征求和，生成图级表示 z = Σ_i α_i h_i。

这种设计使得模型能够自动识别对图表示贡献较大的关键节点。在CFG中，包含敏感API调用或复杂控制流的节点往往获得更高的注意力权重。在FCG中，度中心性高的核心函数节点会被赋予更大权重。实验表明，注意力池化相比平均池化，检测性能提升约2-3个百分点。

**算法6 注意力池化算法**

| **算法6** 注意力池化 |
|:---|
| **输入** 节点嵌入 H ∈ R^(N×d)，批次索引 batch ∈ Z^N |
| **输出** 图级嵌入 Z ∈ R^(B×d) |
| 1. 初始化注意力权重矩阵 W_a ∈ R^(d×1) |
| 2. 计算注意力分数 E ← H · W_a |
| 3. 应用sigmoid归一化 A ← σ(E) |
| 4. 加权节点特征 H_weighted ← H ⊙ A |
| 5. 按批次聚合 Z ← GlobalAddPool(H_weighted, batch) |
| 6. **return** Z |

算法第2行通过矩阵乘法计算所有节点的注意力分数，时间复杂度为 O(Nd)。第3行的sigmoid函数将分数映射到 (0, 1) 区间，作为注意力权重。第4行执行逐元素乘法，为每个节点特征加权。第5行使用全局加和池化按批次聚合节点特征，生成每个图的表示向量。

注意力池化层的参数量仅为 d 个（权重向量 W_a），相比节点特征维度可忽略不计。这种轻量级设计在引入自适应聚合能力的同时，几乎不增加模型复杂度。训练过程中，注意力权重通过反向传播自动学习，无需人工设计或调整。

实验分析表明，在CFG中，注意力权重与节点的圈复杂度和敏感API调用呈正相关。在FCG中，入度和出度较高的函数节点获得更高权重。这验证了注意力机制能够自动捕捉对恶意代码检测最关键的结构特征，为模型的可解释性提供了支持。
