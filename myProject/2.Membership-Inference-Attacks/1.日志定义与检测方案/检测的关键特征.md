#### 用于检测成员推断攻击的关键特征

1. 特征：高密度查询聚类 (High-Density Query Clustering)
这是攻击最显著的外部行为。攻击者为了训练攻击模型，需要围绕一个敏感样本生成大量极其相似的数据点进行查询 。
    检测信号:
        输入向量高度相似: 对单个任务内提交的所有模型查询请求（Query 1/151, Query 2/151, ... ）的输入特征向量进行分析。这些向量的欧氏距离或余弦距离会非常小，即平均相似度极高 。
低方差输入参数: 任务中绝大部分查询请求的输入参数值都集中在一个非常狭窄的范围内，方差极低。这与正常科研任务中输入数据具有多样性的特点形成鲜明对比 。

2. 特征：寄生模型训练 (Parasitic Model Training)
这是攻击的核心意图。攻击任务本身包含了一个“模型中的模型”，它消费平台模型的预测结果，用于训练一个本地的、恶意的攻击模型 。
    检测信号:
        包含模型训练逻辑: 通过静态或动态代码分析，检测到任务代码中包含了机器学习模型的训练调用，例如 attack_model.fit(...) 。这是成员推断攻击的决定性证据 。
加载可疑库: 任务运行时加载了如 sklearn 等机器学习库，但其主要功能本应是数据查询和分析，而不是模型训练 。
日志明确指示: 任务的执行日志中出现了明确的训练提示，如 [+] Training the membership inference attack model... 。

3. 特征：模型响应的二元性 (Binarity of Model Response)
攻击者不关心模型预测的科学意义，只关心模型对“成员”和“非成员”的“反应”差异，尤其是置信度 。
    检测信号:
        置信度离群: 在任务的所有查询结果中，模型返回的置信度（Confidence）存在明显的离群值。如日志所示，敏感样本的置信度为 0.9876 ，而大量影子数据的置信度则分布在 0.4 到 0.6 的较低区间 。
高方差置信度: 整个任务的置信度序列呈现出极高的方差，或呈双峰分布。日志中的警告信息 WARNING: High variance detected in model confidence scores 就是一个直接的信号。
代码逻辑简单: 任务代码在获取模型预测结果后，只提取了置信度等少量元数据，而对详细的预测内容本身没有进行复杂的后续分析或使用 。

4. 特征：输入/输出信息量严重失衡 (Drastic I/O Imbalance)
攻击任务的行为模式是“海量查询，少量输出”。它向平台索取了大量信息（模型对上百个数据点的反应），但其最终产出的价值信息却极少 。
    检测信号:
        极低的“输出/请求”比: 任务的“最终输出数据字节数”与“模型查询总次数”的比值异常低 。例如，日志显示共进行了 151 次查询 ，但最终的 membership_inference_result.log 文件只记录了一个分数和一个布尔值 。
输出内容简单: 最终的输出结果不是复杂的数据集或图表，而仅仅是一个简单的判断性结论，如 "Conclusion: The sample is LIKELY a member..." 。

5. 特征：运行时动态数据生成 (On-the-Fly Data Generation)
为了制造用于对比的“非成员”数据，攻击任务通常不会加载一个现有的数据集，而是在运行时动态地、算法化地生成查询数据 。
    检测信号:
        存在数据生成函数: 代码中包含明显的数据生成逻辑，如 generate_shadow_data 函数，它通过对单个样本添加随机噪声 (np.random.normal) 来创建大量新数据。
        循环的资源消耗模式: 任务的资源占用（CPU、内存）可能呈现出“计算（生成数据） -> 等待（网络查询） -> 计算 -> 等待”的周期性模式，而非传统任务的“一次性加载数据 -> 持续计算”的模式 。
通过组合监控以上这些特征，您就可以构建一个强大的检测系统，从海量任务中精准地识别出成员推断攻击行为。

1. 查询模式：高密度聚类
    特征：攻击的核心是向模型提交大量在特征空间上高度聚类的查询（一个敏感样本和其生成的众多微小扰动的影子样本）。这与正常用户提交具有多样性的数据进行分析的模式截然不同。
    检测信号：单个任务提交的多个查询请求，其输入特征向量在数值上非常接近（欧氏距离极小或余弦相似度极高），形成一个密集的查询簇。

2. 核心行为：寄生模型训练
    特征：攻击任务的最终目的不是获取预测结果，而是收集模型的“反应”（如置信度），并利用这些数据在任务内部训练一个本地的、二元的“攻击模型”，用以判断成员身份。
    检测信号：任务执行过程中出现了模型训练行为。例如，在日志中记录到attack_model.fit()等调用，或任务的摘要信息中明确标记检测到了内部训练活动。

3. 响应特征：置信度分布异常
    特征：由于模型对训练过的“成员”数据通常会表现出更高的预测置信度，因此，攻击任务获取的置信度序列会呈现出明显的异常分布。
    检测信号：在一个任务收集的所有模型响应中，置信度（Confidence）的方差极大。通常表现为一个或几个极高的离群值（对应敏感成员样本）和大量较低的、集中的值（对应影子非成员样本）。

4. 资源模式：持续查询与微小输出
    特征：攻击任务的生命周期绝大部分消耗在对模型API的密集查询上，其最终产出的有效信息量却极小（通常只是一个概率值和布尔判断）。
    检测信号：任务的“查询速率”（总查询次数/总执行时间）远高于正常科研任务的基线。同时，任务最终状态记录的输出数据量极小。
