# 梯度窃取攻击检测方案文档 (V5.0 高阶混淆版)

## 1. JSON 日志格式说明

在 V5.0 版本中，为了模拟攻击者的高级伪装能力，我们移除了所有显式的“攻击标签”。无论是正常训练还是恶意攻击，其日志结构在**文本层面完全一致**。

### 1.1 数据结构概览

每个样本包含一个 `logs` 列表，列表中的每一项代表一步计算操作。

JSON

```
{
  "session_id": "node_123",
  "label": 1,                 // 0: 正常, 1: 恶意 (Ground Truth)
  "client_type": "malicious_mimic", // 细分类型：伪装攻击者 / 抖动正常节点
  "logs": [
    {
      "timestamp": "2025-11-27 10:00:00,500",
      "phase": "EXECUTION_STEP",  // 【混淆点】统一阶段名，不再区分 TRAIN/ATTACK
      "metrics": {
        "val": 2.1354,            // 【混淆点】统一数值名，代表 Loss，但不再标记类型
        "seq": 1                  // 序列号，总是连续的 (1, 2, 3...)，掩盖了真实步数
      }
    },
    ...
  ]
}
```

### 1.2 字段详解

- **`phase`**: 固定为 `"EXECUTION_STEP"`。攻击者不再使用特殊的 Phase 字符串，导致无法通过 One-Hot 编码直接识别。
- **`metrics.val`**:
  - **正常节点**: 代表预测损失 (Prediction Loss)。数值平滑下降。
  - **恶意节点**: 代表梯度匹配损失 (Gradient Matching Loss)。经过攻击者缩放处理，数值范围与正常节点重叠 (0.0~4.0)，但保留了非凸优化的震荡特征。
- **`metrics.seq`**: 只是日志的流水号。攻击者内部可能跑了 500 步，但对外只打印第 100、200... 步，并将 `seq` 伪造为 1, 2...，导致基于“步数跳跃”的检测失效。

## 2. 检测的关键特征 (Key Indicators)

由于宏观特征（时长、次数）已失效，本方案利用**“物理指纹”**进行检测。这些特征源于计算机系统底层的物理限制，难以被软件层面的伪装完全消除。

### 2.1 时间方差 (Time Variance)

- **正常节点 (Normal Jittery)**: 受制于真实的网络波动和 CPU 调度，其单步耗时 ($\Delta t$) 服从**正态分布**，方差较大（$\sigma \approx 0.2s$）。
- **恶意节点 (Malicious Mimic)**: 为了凑够正常节点的时长，攻击者使用机器控制的 `sleep()`。这种延时的精度极高，导致其单步耗时服从**均匀分布**或接近固定值，方差极小（$\sigma \approx 0.01s$）。
- **检测原理**: **方差差异**。也就是“过分完美的节奏”反而是可疑的。

### 2.2 损失锯齿度 (Loss Jaggedness)

- **正常节点**: 训练数据的 Loss 通常沿着梯度方向平滑下降。
- **恶意节点**: 梯度反演是一个从随机噪声恢复数据的过程（非凸优化）。即使攻击者对输出数值进行了平滑处理，其 Loss 轨迹仍会表现出**局部的剧烈震荡**和“回跳”现象。
- **检测原理**: **二阶导数异常**。

## 3. 特征提取方案设计

我们采用 **“瞬时特征 + 全局统计特征”** 的混合编码方式。我们将整个 Session 的统计属性“广播”到每一个时间步上，使 LSTM 既能看到当前状态，又能感知全局分布。

### 3.1 提取流程

1. **预计算全局统计量**: 扫描整个 Session 的日志，计算出均值、标准差、斜率等 6 个统计指标。
2. **计算瞬时量**: 遍历每条日志，计算当前的耗时和 Loss。
3. **向量拼接**: 将 `[瞬时特征]` 与 `[全局特征]` 拼接，形成最终的特征向量。
4. **噪声注入**: 在最终特征上叠加微量高斯噪声，模拟真实环境的测量误差，防止模型训练出 100% 准确率。

### 3.2 为什么这样设计？

- 如果只用瞬时特征（如当前 Loss），模型无法判断这个 Loss 是偏高还是偏低（因为没有上下文）。
- 如果只用全局特征，模型就退化成了传统机器学习（随机森林），失去了 LSTM 处理时序变化的能力。
- **混合编码**让 LSTM 可以学习到：“在方差很小（全局特征）的背景下，出现了一次剧烈的 Loss 跳变（瞬时特征），这大概率是攻击。”
