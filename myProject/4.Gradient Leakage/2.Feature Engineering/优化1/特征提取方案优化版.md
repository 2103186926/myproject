# 针对结构化日志的梯度窃取检测方案 (v2.0 - 增强鲁棒性版)

## 1. 方案背景与演进

### 1.1 原方案缺陷 (v1.0)

在早期的设计中，我们过于依赖显式的攻击特征（如日志中出现 `ATTACK` 关键字、`gradient_loss` 类型标记）。这种方案在“白盒”模拟中表现完美，但在真实对抗环境（Adversarial Environment）中极为脆弱，因为聪明的攻击者会伪装日志格式。

### 1.2 改进思路 (v2.0)

本方案移除了所有“上帝视角”的标签特征。我们将日志视为一种**行为流（Behavior Stream）**。模型不再通过“读懂日志说了什么”来判断，而是通过分析“日志产生的节奏、密度和数值轨迹”来检测异常。

**核心假设：** 无论攻击者如何修改日志文本，其底层的物理计算过程（大量迭代优化、高频计算、文件落地）很难完全掩盖。

## 2. 数据增强策略 (Data Augmentation)

为了防止模型过拟合“简单特征”（如仅仅根据时长判断），我们在数据集中引入了**模糊样本**：

| **样本类型**             | **占比** | **行为特征**                                   | **挑战点**                                                   |
| ------------------------ | -------- | ---------------------------------------------- | ------------------------------------------------------------ |
| **Normal (Fast)**        | 50%      | Epoch少 (1-5)，耗时短，无IO。                  | 基准样本。                                                   |
| **Normal (Straggler)**   | 20%      | Epoch少，但**单步耗时极长**，总时长久。        | 容易被误判为恶意（如果仅看时长）。需要模型识别出步数很少。   |
| **Malicious (Stealthy)** | 30%      | Iteration多 (100+)，伪装成 Normal 的日志格式。 | 很难区分。需要模型识别出**极高的步数**和**特定的Loss分布**。 |

## 3. 特征工程详细设计 (11维/步)

每条日志被转化为一个 11 维的数值向量。

### 3.1 动力学特征 (Dynamics) - 4维

这些特征捕捉“计算过程”的物理属性。

1. **`delta_time` ($\Delta t$)**:
   - **含义**: 这一步花了多长时间。
   - **对抗意义**: 区分计算密集型步骤（短 $\Delta t$）和 IO/网络等待步骤（长 $\Delta t$）。Normal Straggler 会有均匀的长 $\Delta t$，而攻击者可能是密集的短 $\Delta t$。
2. **`cumulative_time` ($T_{cum}$)**:
   - **含义**: 任务开始多久了。
   - **对抗意义**: 辅助 LSTM 建立时间尺度感。
3. **`log_density` (Hz)**:
   - **含义**: $1 / \Delta t$。
   - **对抗意义**: 攻击者的优化循环往往会在极短时间内产生大量日志（高频爆发），而正常训练受限于数据加载，频率较低。
4. **`step_counter` (Normalized)**:
   - **含义**: 当前 Session 的第几条日志 / 500。
   - **对抗意义**: **这是最强的隐性特征**。如果一个 Session 走到 `step_counter = 0.8` (即400步)，大概率是恶意攻击，因为正常训练很少跑几百个 Epoch。

### 3.2 数值特征 (Metrics) - 1维

1. **`loss_value`**:
   - **含义**: 提取出的数值。
   - **对抗意义**: 即使攻击者不标记 `loss_type`，Gradient Matching Loss 的**收敛数值范围**和**震荡形态**（Variance）通常与 Prediction Loss 不同。LSTM 擅长捕捉这种序列波动的指纹。

### 3.3 敏感行为特征 (Heuristics) - 1维

1. **`is_file_io`**:
   - **含义**: 日志消息中是否包含 `.npy`, `dump`, `save` 等文件落地关键词。
   - **对抗意义**: 梯度窃取通常需要保存重构出的图片/数据。这是一个高风险的“侧信道”特征。

### 3.4 状态特征 (State) - 5维

7-11. phase_ONEHOT:

\* 我们统一了相位名称。正常训练叫 TRAINING_STEP，攻击迭代也叫 TRAINING_STEP。

\* 这迫使 LSTM 不能依赖 ATTACK 这种字符串，而必须依赖 TRAINING_STEP 出现的次数和频率。

## 4. 总结

新的特征方案构建了一个更复杂的分类边界：

- **黑（恶意）**：高频日志 + 步数极多 + 存在文件写入 + Loss 震荡。
- **白（正常）**：低/中频日志 + 步数少 + 无文件写入。
- **灰（Straggler）**：低频日志（耗时长） + 步数少。

模型现在必须学会：“耗时长不一定是恶意，但**耗时短且步数多**一定是恶意”。这正是我们想要达到的非线性分类能力。
