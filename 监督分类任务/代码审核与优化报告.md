## 二、发现的主要问题

### 2.1 fusion_classifier_models.py 问题清单

#### 问题1: 数据标准化重复问题 ⚠️ 严重
**位置**: `LogisticRegressionClassifier`, `SVMClassifier`

**问题描述**: 
- 模型内部使用 `StandardScaler` 进行标准化
- 但在 `run_comparison_experiments.py` 中数据已经标准化
- 导致数据被标准化两次，影响模型性能

**解决方案**:
```python
# 添加 use_scaler 参数控制是否标准化
def __init__(self, max_iter: int = 1000, random_state: int = 42, use_scaler: bool = True):
    self.use_scaler = use_scaler
```

#### 问题2: DeepLearningTrainer 缺少关键功能 ⚠️ 中等
**问题描述**:
1. 缺少梯度裁剪，可能导致梯度爆炸
2. 早停后未恢复最佳模型权重
3. `predict` 方法未使用批处理，大数据集可能内存溢出
4. 缺少模型保存/加载功能

**解决方案**:
```python
# 1. 添加梯度裁剪
torch.nn.utils.clip_grad_norm_(self.model.parameters(), max_norm=1.0)

# 2. 保存并恢复最佳模型
self.best_model_state = {k: v.cpu().clone() for k, v in self.model.state_dict().items()}
self.model.load_state_dict(self.best_model_state)

# 3. 批处理预测
def predict(self, X: np.ndarray, batch_size: int = 256):
    # 分批处理避免内存溢出

# 4. 添加保存/加载方法
def save_model(self, path: str)
def load_model(self, path: str)
```

#### 问题3: BiLSTM 特征reshape语义问题 ⚠️ 轻微
**问题描述**:
- 将256维融合特征强制reshape为(32, 8)序列
- 可能破坏特征的原始语义关系

**建议**: 
- 在文档中明确说明此问题
- 推荐优先使用MLP或Transformer

#### 问题4: EnsembleClassifier 设计缺陷 ⚠️ 中等
**问题描述**:
- `forward` 方法需要同时传入 `torch.Tensor` 和 `np.ndarray`
- 接口设计不合理，使用不便

**建议**: 
- 简化为纯深度学习融合
- 或将XGBoost预测结果预先计算并作为特征输入

### 2.2 run_comparison_experiments.py 问题清单

#### 问题5: 数据预处理流程问题 ⚠️ 严重
**位置**: `prepare_data` 函数

**问题描述**:
- 函数内部强制标准化
- 但ML模型内部又标准化一次
- 导致重复标准化

**解决方案**:
```python
def prepare_data(..., normalize: bool = True):
    if normalize:
        scaler = StandardScaler()
        X_train = scaler.fit_transform(X_train)
        X_val = scaler.transform(X_val)
        X_test = scaler.transform(X_test)
        return ..., scaler
```

#### 问题6: 缺少异常处理 ⚠️ 中等
**问题描述**:
- 模型训练过程中没有 try-except
- 一个模型失败会导致整个实验中断

**解决方案**:
```python
for model_name, params in ml_models:
    try:
        # 训练代码
    except Exception as e:
        logger.error(f"训练 {model_name} 时出错: {e}")
        continue
```

#### 问题7: 可视化中文乱码 ⚠️ 轻微
**问题描述**:
- matplotlib 默认不支持中文
- 图表标题显示为方框

**解决方案**:
```python
plt.rcParams['font.sans-serif'] = ['SimHei', 'Microsoft YaHei']
plt.rcParams['axes.unicode_minus'] = False
```

#### 问题8: 硬编码输入维度 ⚠️ 轻微
**位置**: `run_dl_comparison` 函数

**问题描述**:
```python
model = ClassifierFactory.create_dl_model(model_name, input_dim=256, **params)
```
- 硬编码为256维，缺乏灵活性

**解决方案**:
```python
input_dim = X_train.shape[1]
model = ClassifierFactory.create_dl_model(model_name, input_dim=input_dim, **params)
```

## 六、使用建议

### 6.1 数据预处理建议
```python
# 推荐做法：在prepare_data中统一标准化
X_train, X_val, X_test, y_train, y_val, y_test, scaler = prepare_data(
    X, y, normalize=True
)

# 创建模型时不再标准化

    'logistic_regression', 
    use_scaler=False  # 数据已标准化
)
```

### 6.2 模型选择建议
- **小数据集(<1000样本)**: 推荐 Logistic Regression, SVM
- **中等数据集(1000-10000)**: 推荐 Random Forest, XGBoost, LightGBM
- **大数据集(>10000)**: 推荐 MLP, Transformer
- **序列特征**: 推荐 BiLSTM, Transformer
- **融合特征**: 推荐 MLP, XGBoost

### 6.3 超参数调优建议
```python
# XGBoost/LightGBM
- n_estimators: 100-500
- max_depth: 3-10
- learning_rate: 0.01-0.1

# 深度学习
- learning_rate: 0.0001-0.01
- batch_size: 16-128
- epochs: 50-200
- patience: 10-20
```
