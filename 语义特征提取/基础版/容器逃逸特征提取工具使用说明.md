# 容器逃逸特征提取工具使用说明

## 概述

`semantic_feature_extractor.py` 已被改造为专门针对**Python容器逃逸静态检测**的语义特征提取工具。该工具基于《Python容器逃逸静态检测规则手册》，能够从AST JSON文件中提取128维的容器逃逸语义特征向量，并生成详细的风险评估报告。

## 主要功能

### 1. 容器逃逸语义特征提取（128维）

特征向量分为三个部分：

#### 第一部分：容器逃逸敏感函数频次特征（32维）
- **system_commands**（系统命令执行）：os.system, subprocess.*, nsenter, mount
- **process_control**（进程控制）：os.fork, os.dup2, pty.spawn
- **dangerous_libs**（危险库调用）：ctypes.CDLL, docker.from_env, kubernetes.client
- **network_recon**（网络侦察）：socket.connect, requests.get（元数据服务）
- **file_access**（文件访问）：open（敏感路径）
- **kernel_syscalls**（内核系统调用）：setns, unshare, mount, ptrace
- **container_runtime**（容器运行时）：docker/containerd相关操作
- **k8s_operations**（K8s操作）：kubernetes API调用

每个类别占用4个特征维度：
1. 是否存在该类别的调用（0/1）
2. 调用次数（对数缩放）
3. 相对频率
4. 重要性权重（平方项）

#### 第二部分：攻击模式序列特征（64维）

覆盖16个攻击模式类别，每类4个特征：
- **cmd_execution**：系统命令执行
- **file_fingerprint**：容器环境指纹探测
- **file_escape**：容器逃逸文件路径
- **lib_dangerous**：危险库调用
- **lib_container**：容器相关库
- **net_recon**：网络侦察
- **exploit_kernel**：内核漏洞利用（Dirty COW, Dirty Pipe等）
- **exploit_runtime**：运行时漏洞利用（runc, cgroup等）
- **k8s_privilege**：K8s特权配置
- **k8s_persistence**：K8s持久化
- **cred_theft**：凭证窃取
- **dos_attack**：DoS攻击

#### 第三部分：路径和配置特征（32维）

检测8类敏感路径访问：
- **container_runtime_paths**：/var/run/docker.sock等
- **proc_sensitive_paths**：/proc/1/, /proc/self/exe等
- **cgroup_paths**：/sys/fs/cgroup, release_agent等
- **k8s_secret_paths**：K8s凭证路径
- **host_mount_paths**：宿主机挂载点
- **kernel_config_paths**：内核配置路径
- **container_fingerprint_paths**：容器指纹路径
- **credential_paths**：凭证文件路径

### 2. 容器逃逸风险评估

基于检测规则手册5.2节的规则集，自动评估代码的容器逃逸风险：

#### 风险等级
- **CRITICAL**（严重）：发现严重的容器逃逸威胁
- **HIGH**（高）：发现高风险的逃逸行为
- **MEDIUM**（中）：发现中等风险
- **LOW**（低）：发现低风险
- **SAFE**（安全）：未发现明显风险

#### 检测规则
- **CE-CMD-01/02/03**：系统命令执行检测
- **CE-FILE-01/02/03/04**：敏感文件路径访问检测
- **CE-LIB-01/02/03/04**：危险库调用检测
- **CE-NET-01/02/03**：网络行为检测

### 3. 代码混淆检测

保留原有的代码混淆检测功能，识别：
- 高熵值代码
- 超长代码行
- base64编码
- eval/exec使用
- 变量名混淆
- 字符串拼接

## 使用方法

### 基本用法

```bash
python semantic_feature_extractor.py <ast_json_file>
```

### 完整参数

```bash
python semantic_feature_extractor.py sample.ast.json \
    -r high_risk_code.py \
    -o output_features.npy \
    --no-report
```

### 参数说明

- `ast_json_file`：必需，AST JSON文件路径
- `-c, --cfg`：可选，CFG元数据文件路径（保留兼容性）
- `-r, --high_risk`：可选，高危测试文件路径（用于增强检测）
- `-o, --output`：可选，输出文件路径（默认：`<basename>_container_escape_features.npy`）
- `--no-report`：可选，不生成风险评估报告

## 输出文件

### 1. 特征向量文件（.npy）

NumPy格式的128维特征向量，可用于机器学习模型训练或推理。

```python
import numpy as np
features = np.load('output_features.npy')
print(features.shape)  # (128,)
```

## 工作流程

```
Python源代码
    ↓
AST解析
    ↓
语义特征提取
    ├─→ 敏感函数频次特征 (32维)
    ├─→ 攻击模式序列特征 (64维)
    └─→ 路径配置特征 (32维)
    ↓
特征向量合并 (128维)
    ↓
    ├─→ 特征向量保存 (.npy文件)
    └─→ 风险评估报告 (.json文件)
```

## 应用场景

### 1. 静态代码安全扫描
在CI/CD流程中集成，自动检测用户提交的Python代码是否存在容器逃逸风险。

### 2. 机器学习模型训练
使用提取的128维特征向量训练容器逃逸检测模型。

### 3. 安全审计
对海洋科学云平台等容器化环境中的用户代码进行安全审计。

### 4. 威胁情报分析
分析已知的容器逃逸攻击样本，提取特征用于威胁检测。

## 检测覆盖范围

### 攻击阶段覆盖

1. **侦察阶段（Reconnaissance）**
   - 环境与身份指纹探测
   - 敏感配置与挂载点探测
   - 云原生API与元数据探测
   - 网络与服务网格探测

2. **逃逸阶段（Escape）**
   - 内核漏洞利用（Dirty COW, Dirty Pipe, Polkit等）
   - 容器运行时漏洞（runc, containerd-shim等）
   - 危险配置利用（Docker Socket, Cgroup, Device等）
   - 特殊场景逃逸（Kubelet日志, Ptrace注入等）

3. **持久化阶段（Persistence）**
   - K8s恶意资源部署
   - 凭证窃取与横向移动
   - Webshell植入

4. **破坏阶段（Destruction）**
   - 反弹Shell
   - 资源耗尽型DoS攻击

## 技术特点

1. **规则驱动**：基于《Python容器逃逸静态检测规则手册》的标准化规则
2. **多维特征**：从函数调用、攻击模式、路径访问三个维度提取特征
3. **风险量化**：自动计算风险分数并分级
4. **可扩展性**：易于添加新的检测规则和模式
5. **兼容性**：保留原有接口，向后兼容

## 注意事项

1. 该工具进行**静态分析**，不执行代码
2. 需要先将Python源代码转换为AST JSON格式
3. 建议配合源代码文件使用，以提高检测准确率
4. 风险评估基于模式匹配，可能存在误报或漏报
5. 应结合人工审计进行最终判断
