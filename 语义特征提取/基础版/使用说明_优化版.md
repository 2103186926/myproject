### 2. 海洋科学云平台代码审查

```python
# 平台代码审查脚本
import subprocess
import json
import sys

def scan_user_code(code_file):
    """扫描用户提交的代码"""
    # 运行特征提取
    result = subprocess.run(
        ['python', 'semantic_feature_extractor.py', code_file],
        capture_output=True,
        text=True
    )
    
    # 读取风险报告
    report_file = code_file.replace('.py', '_container_escape_features_risk_report.json')
    with open(report_file, 'r', encoding='utf-8') as f:
        report = json.load(f)
    
    # 判断是否允许执行
    if report['risk_level'] in ['CRITICAL', 'HIGH']:
        print(f"❌ 代码存在容器逃逸风险，禁止执行！")
        print(f"   风险等级: {report['risk_level']}")
        print(f"   {report['summary']}")
        return False
    
    print(f"✅ 代码安全检查通过")
    return True

# 使用示例
if __name__ == '__main__':
    if not scan_user_code('user_submitted_code.py'):
        sys.exit(1)
```

### 3. LSTM模型训练

```python
import numpy as np
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import LSTM, Dense, Dropout

# 加载训练数据
X_train = []
y_train = []

# 批量加载特征向量
for sample_file, label in training_samples:
    features = np.load(sample_file)
    X_train.append(features)
    y_train.append(label)  # 0=安全, 1=逃逸

X_train = np.array(X_train)
y_train = np.array(y_train)

# 重塑为LSTM输入格式 (samples, timesteps, features)
X_train = X_train.reshape((X_train.shape[0], 1, 128))

# 构建LSTM模型
model = Sequential([
    LSTM(64, input_shape=(1, 128), return_sequences=True),
    Dropout(0.2),
    LSTM(32),
    Dropout(0.2),
    Dense(16, activation='relu'),
    Dense(1, activation='sigmoid')
])

model.compile(
    optimizer='adam',
    loss='binary_crossentropy',
    metrics=['accuracy']
)

# 训练模型
model.fit(X_train, y_train, epochs=50, batch_size=32, validation_split=0.2)

# 预测新样本
new_features = np.load('new_sample_container_escape_features.npy')
new_features = new_features.reshape((1, 1, 128))
prediction = model.predict(new_features)
print(f"逃逸概率: {prediction[0][0]:.2%}")
```

## 技术实现

### AST直接解析

```python
import ast

# 解析Python源代码
with open('source.py', 'r') as f:
    source_code = f.read()

# 生成AST
tree = ast.parse(source_code)

# 遍历AST节点
for node in ast.walk(tree):
    if isinstance(node, ast.Call):
        # 提取函数调用信息
        func_name = extract_function_name(node)
        if is_sensitive(func_name):
            print(f"发现敏感函数: {func_name}")
```

### 特征提取流程

```
Python源代码
    ↓
AST解析 (ast.parse)
    ↓
节点遍历 (ast.walk)
    ↓
特征提取
    ├─→ 敏感函数频次 (32维)
    ├─→ 攻击模式匹配 (64维)
    └─→ 路径配置检测 (32维)
    ↓
特征向量 (128维)
    ↓

    ↓

```

## 常见问题

### Q1: 支持哪些Python版本？
A: 支持Python 3.6及以上版本。

### Q2: 如何处理语法错误的代码？
A: 工具会自动检测语法错误并给出友好提示，不会崩溃。

### Q3: 可以分析加密或混淆的代码吗？
A: 可以检测混淆特征，但无法分析加密后的代码。建议先解密。

### Q4: 批量处理大量文件时如何提高效率？
A: 使用多进程并行处理：

```python
from multiprocessing import Pool

def process_file(file):
    subprocess.run(['python', 'semantic_feature_extractor.py', file])

if __name__ == '__main__':
    files = glob.glob('*.py')
    with Pool(4) as p:
        p.map(process_file, files)
```

### Q5: 如何自定义检测规则？
A: 编辑 `semantic_feature_extractor.py` 中的以下常量：
- `CONTAINER_ESCAPE_MODULES`: 添加敏感模块
- `CONTAINER_ESCAPE_PATTERNS`: 添加正则表达式模式
- `ESCAPE_RISK_RULES`: 调整风险等级
