# semantic_feature_extractor.py 优化总结

## 优化目标

将工具从**基于AST JSON文件**的特征提取改造为**直接从Python源代码**提取特征，简化使用流程。

## 核心改动

### 1. 移除中间文件依赖

#### 优化前
```python
class ContainerEscapeFeatureExtractor:
    def __init__(self, ast_json_file: str, cfg_meta_file: Optional[str] = None, 
                 high_risk_file: Optional[str] = None):
        # 需要加载AST JSON文件
```

#### 优化后 ✨
```python
class ContainerEscapeFeatureExtractor:
    def __init__(self, source_file: str):
        # 直接加载并解析Python源代码
        with open(source_file, 'r', encoding='utf-8') as f:
            self.source_code = f.read()
        
        # 使用Python内置ast模块解析
        self.ast_tree = ast.parse(self.source_code, filename=source_file)
```

```python
def _find_sensitive_calls(self, root_node: Any) -> List[str]:
    sensitive_calls = []
    for node in ast.walk(root_node):  # 简洁高效
        if isinstance(node, ast.Call):
            func_name = self._get_function_name_from_ast(node)
            if is_sensitive(func_name):
                sensitive_calls.append(func_name)
    return sensitive_calls
```

### 3. 简化函数名提取逻辑

#### 优化前：复杂的字典解析
```python
def _get_function_name(self, call_node: Dict[str, Any]) -> Optional[str]:
    # 策略1: 检查children
    children = call_node.get("children", [])
    if children and len(children) > 0:
        first_child = children[0]
        if isinstance(first_child, dict):
            if first_child.get("type") == "Name" and "id" in first_child:
                return first_child["id"]
            # ... 更多复杂逻辑
    
    # 策略2: 检查func字段
    # 策略3: 遍历所有子节点
    # ... 100+行代码
```

#### 优化后：利用AST节点属性 ✨
```python
def _get_function_name_from_ast(self, call_node: ast.Call) -> Optional[str]:
    func = call_node.func
    
    # 简单函数调用：func()
    if isinstance(func, ast.Name):
        return func.id
    
    # 属性调用：module.func()
    elif isinstance(func, ast.Attribute):
        parts = []
        current = func
        while isinstance(current, ast.Attribute):
            parts.append(current.attr)
            current = current.value
        if isinstance(current, ast.Name):
            parts.append(current.id)
            parts.reverse()
            return '.'.join(parts)
    
    return None
```

### 4. 移除不必要的代码

#### 删除的方法

- 所有与JSON AST相关的解析逻辑

#### 删除的参数
- `cfg_meta_file` - CFG元数据文件（不再需要）
- `high_risk_file` - 高危测试文件（不再需要）
- `ast_json_file` - AST JSON文件（替换为source_file）
