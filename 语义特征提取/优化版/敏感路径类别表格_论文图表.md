# 敏感路径类别表格（论文第3章图表补充）

## 表 3-X：容器逃逸检测中的敏感路径类别定义

| 类别编号 | 类别名称 | 类别说明 | 典型路径模式 | 安全风险 |
|---------|---------|---------|------------|---------|
| 1 | 容器运行时接口<br>(Container Runtime) | 容器运行时的Socket接口文件，攻击者可通过这些接口直接控制容器引擎 | `/var/run/docker.sock`<br>`/var/run/containerd.sock`<br>`/run/containerd`<br>`/var/run/crio` | 高危：可直接操作宿主机容器，实现完全逃逸 |
| 2 | 进程敏感信息<br>(Proc Sensitive) | Linux进程文件系统中的敏感路径，包含进程内存、可执行文件等关键信息 | `/proc/1/`<br>`/proc/self/exe`<br>`/proc/self/mem`<br>`/proc/self/fd/`<br>`/proc/[pid]/exe`<br>`/proc/[pid]/mem` | 高危：可读取宿主机进程信息，修改进程内存 |
| 3 | Cgroup与Sysfs<br>(Cgroup & Sysfs) | Linux控制组和系统文件系统，用于资源隔离和内核参数配置 | `/sys/fs/cgroup`<br>`/sys/kernel/`<br>`release_agent`<br>`notify_on_release`<br>`devices.allow` | 高危：可突破资源限制，执行宿主机命令 |
| 4 | K8s与云凭证<br>(K8s & Cloud Credentials) | Kubernetes服务账户令牌和云平台凭证文件 | `/var/run/secrets/kubernetes`<br>`.kube/config`<br>`.aws/credentials`<br>`.config/gcloud` | 高危：可获取集群权限，访问云资源 |
| 5 | 宿主机文件系统<br>(Host Filesystem) | 挂载的宿主机文件系统路径 | `/host/`<br>`/hostfs/`<br>`/proc/1/root`<br>`hostPath`<br>`volumeMounts` | 高危：可直接访问宿主机文件系统 |
| 6 | 设备文件<br>(Device Files) | 物理设备和块设备文件 | `/dev/mem`<br>`/dev/kmem`<br>`/dev/sda`<br>`/dev/vda`<br>`/dev/nvme` | 高危：可直接访问物理内存和磁盘 |

## 表 3-Y：敏感路径类别特征提取方法

| 特征维度 | 特征名称 | 计算方法 | 取值范围 | 特征意义 |
|---------|---------|---------|---------|---------|
| 1 | 类别存在性 | 该类别是否出现 | {0, 1} | 二值特征，标识是否访问该类敏感路径 |
| 2 | 访问频次（对数） | log(1 + 访问次数) | [0, +∞) | 反映访问强度，对数变换降低异常值影响 |
| 3 | 模式覆盖率 | 匹配模式数 / 总模式数 | [0, 1] | 反映攻击手法的多样性 |
| 4 | 最大单模式频次（对数） | log(1 + max(单个模式次数)) | [0, +∞) | 识别集中攻击特定路径的行为 |

**说明**：每个敏感路径类别提取4维特征，共6类 × 4维 = 24维特征向量。

## 表 3-Z：敏感路径类别检测示例

| 代码片段 | 检测到的类别 | 匹配的路径模式 | 风险评估 |
|---------|------------|--------------|---------|
| `open('/var/run/docker.sock')` | 容器运行时接口 | `/var/run/docker.sock` | 高危：尝试访问Docker Socket |
| `os.system('cat /proc/1/environ')` | 进程敏感信息 | `/proc/1/` | 高危：读取宿主机init进程环境变量 |

| `dd if=/dev/mem of=dump.bin` | 设备文件 | `/dev/mem` | 高危：读取物理内存 |

## 特征提取算法流程

```
输入：Python源代码字符串 code
输出：24维敏感路径特征向量 features

1. 初始化特征向量 features = zeros(24)
2. 定义6个敏感路径类别及其正则表达式模式
3. For each 类别 i in [0, 5]:
   a. 初始化 category_count = 0, matched_patterns = 0, max_count = 0
   b. For each 模式 pattern in 类别[i]:
      - 使用正则表达式在代码中查找匹配
      - 累加匹配次数到 category_count
      - 如果匹配次数 > 0，matched_patterns += 1
      - 更新 max_count = max(max_count, 当前匹配次数)
   c. 计算该类别的4维特征：
      - features[i*4] = 1 if category_count > 0 else 0
      - features[i*4+1] = log(1 + category_count)
      - features[i*4+2] = matched_patterns / 模式总数
      - features[i*4+3] = log(1 + max_count)
4. 返回 features
```
