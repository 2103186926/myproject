## 主要改进

### 1. 数据量大幅提升
**旧版本问题：**
- V1/V2 只能生成固定的 15 个样本（8 正常 + 7 恶意）
- 数据量太少，无法满足 GNN 训练的 batch_size 要求
- 导致每个 batch 中样本数量不足

**V3 解决方案：**
- 默认生成 100 个样本（50 正常 + 50 恶意）
- 支持自定义数量，可生成数百甚至上千个样本
- 完全满足深度学习模型对数据量的要求

### 2. 样本多样性和独立性
**旧版本问题：**
- 只有 2-3 个固定模板
- 通过简单修改参数生成变体，相似度高
- 样本之间缺乏区分性

**V3 解决方案：**
- 20+ 种海洋科学计算场景模板
- 每个样本独立生成，使用不同的函数组合
- 函数数量随机（5-15 个），确保多样性
- 函数调用顺序随机化
- 样本之间有明显差异，适合 GNN 学习

### 3. 函数调用关系更复杂
**旧版本问题：**
- 函数调用关系简单
- 难以体现 FCG 的复杂性

**V3 解决方案：**
- 每个样本包含 5-15 个函数（随机）
- 80% 类成员函数 + 20% 独立函数
- main 函数调用所有定义的函数，形成完整的 FCG
- 每个函数内部包含复杂的控制流（循环、条件、异常处理）
- 充分体现 CFG-GNN 和 FCG-GNN 的学习能力

### 4. 恶意代码类型更丰富
**旧版本问题：**
- 只有 3 种恶意操作类型
- 恶意代码模式单一

**V3 解决方案：**
- 10 种容器逃逸攻击类型
- 恶意操作分散在不同函数中（30% 的函数包含恶意代码）
- 更真实地模拟实际攻击场景

### 5. 自动化程度更高
**V3 新增功能：**
- 自动生成 `labels.csv` 标签文件
- 自动生成 `statistics.json` 统计信息
- 提供完整的训练命令示例
- 支持随机种子设置，保证可重复性


### 新版本使用方式
```bash
# V3: 灵活生成任意数量的样本

python test_case_generator_v3.py

# 生成 200 个样本
python test_case_generator_v3.py --total 200

# 生成 500 个样本到指定目录
python test_case_generator_v3.py --total 500 --output_dir ./large_dataset

# 使用特定随机种子
python test_case_generator_v3.py --total 100 --seed 123
```

```bash
    --source_dir ./test_cases \

    --source_dir ./test_cases \
```

## 注意事项

1. **删除旧数据**：如果之前使用 V1/V2 生成过数据，建议删除后重新生成
2. **标签文件**：V3 自动生成的 labels.csv 格式与训练脚本完全兼容
3. **磁盘空间**：生成大量样本时注意磁盘空间（1000 个样本约 50-100 MB）
4. **随机种子**：使用相同的 seed 可以生成相同的数据集，便于实验重复
