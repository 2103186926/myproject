### 问题 #2: 验证集为空导致除零错误 ✅

**日期**：2026-02-05  
**文件**：`gnn_trainer_v2.py`  
**错误信息**：
```
ZeroDivisionError: float division by zero
```

**原因**：
- 数据集太小（仅6个样本），划分后FCG验证集为空
- `_validate` 方法中计算平均损失时除以 `len(val_loader)`，导致除零

**解决方案**：

1. **在 `_validate` 方法中添加空验证集检查**（第488-490行）：
```python
def _validate(self, model, val_loader, criterion):
    """验证模型"""
    model.eval()
    val_loss = 0.0
    
    # 如果验证集为空，返回0
    if len(val_loader) == 0:
        return 0.0
    
    # ... 其余代码
```

2. **在训练方法中添加验证集检查**：
```python
# train_cfg_gnn 方法（第395行）
if val_loader is not None and len(val_loader) > 0:
    val_loss = self._validate(model, val_loader, criterion)
    # ...

# train_fcg_gnn 方法（第457行）
if val_loader is not None and len(val_loader) > 0:
    val_loss = self._validate(model, val_loader, criterion)
    # ...
```

### 问题 #3: FCG模型输入维度不匹配 ✅

**日期**：2026-02-05  
**文件**：`main_workflow_v2.py`  
**错误信息**：
```
RuntimeError: mat1 and mat2 shapes cannot be multiplied (10x128 and 256x1024)
```

**原因**：
- **训练时**：FCG数据特征被增强为256维（128维原始 + 128维CFG嵌入）
- **测试时**：FCG数据特征未被增强，仍为128维
- **模型期望**：FCG模型输入层期望256维特征
- 导致维度不匹配错误

**详细分析**：

1. **训练阶段**（`train_models` 函数，第323-329行）：
```python
# 为FCG节点添加CFG嵌入
cfg_emb_dim = config.model.cfg_out_channels  # 128
num_nodes = fcg_data.x.size(0)
cfg_embeddings = torch.zeros(num_nodes, cfg_emb_dim)
fcg_data.x = torch.cat([fcg_data.x, cfg_embeddings], dim=1)  # 256维
```

2. **测试阶段**（原 `evaluate_models` 函数）：
```python
# 直接创建DataLoader，没有特征增强
test_loader = DataLoader(test_data, batch_size=config.training.batch_size)
```

3. **FCG模型定义**（`gnn_trainer_v2.py` 第145行）：
```python
# FCG节点特征 = 原始特征 + CFG嵌入
in_channels = config.model.node_feature_dim + config.model.cfg_out_channels
# 128 + 128 = 256
```

**解决方案**：

在 `evaluate_models` 函数中添加特征增强逻辑（第360-380行）：

```python
def evaluate_models(cfg_model: CFGGNN_V2,
                   fcg_model: FCGGNN_V2,
                   test_data: List[Data],
                   config: Config,
                   output_dir: str,
                   logger: logging.Logger) -> Dict:
    """评估模型"""
    logger.info("=" * 60)
    logger.info("阶段4: 模型评估")
    logger.info("=" * 60)
    
    # 为测试数据的FCG节点增强特征（与训练时保持一致）
    logger.info("为测试数据增强FCG特征...")
    cfg_emb_dim = config.model.cfg_out_channels  # 128维
    
    for fcg_data in test_data:
        num_nodes = fcg_data.x.size(0)
        # 创建零向量作为CFG嵌入（简化处理）
        cfg_embeddings = torch.zeros(num_nodes, cfg_emb_dim)
        # 拼接特征：原始128维 + CFG嵌入128维 = 256维
        fcg_data.x = torch.cat([fcg_data.x, cfg_embeddings], dim=1)
    
    logger.info(f"测试数据特征维度已增强为: {test_data[0].x.size(1)}维")
    
    test_loader = DataLoader(test_data, batch_size=config.training.batch_size)
    
    evaluator = ModelEvaluator()
    
    # 评估FCG模型（最终模型）
    logger.info("\n评估FCG-GNN模型...")
    results = evaluator.generate_report(fcg_model, test_loader, output_dir)
    
    return results
```

**关键改进**：
1. 在创建 `test_loader` 之前增强FCG特征
2. 确保测试数据维度（256维）与训练时一致
3. 添加日志输出，便于调试和验证

**状态**：✅ 已修复

### 问题 #4: 二分类标签格式不匹配 ✅

**日期**：2026-02-05  
**文件**：`model_evaluator.py`  
**错误信息**：
```
ValueError: Classification metrics can't handle a mix of binary and multilabel-indicator targets
```

**原因**：
- 模型输出 `logits` 形状为 `[batch_size, 2]`（二分类的两个类别）
- 使用 `torch.sigmoid(logits)` 后得到 `[batch_size, 2]`
- 使用 `(probs > 0.5).long()` 得到 `[batch_size, 2]` 的多标签格式
- 而真实标签 `batch.y` 是 `[batch_size]` 的单标签格式
- sklearn的分类指标无法处理格式不一致的预测和标签

**详细分析**：

这是典型的**分类任务类型混淆**问题：
- 错误地将**单标签二分类**当作**多标签分类**处理
- 使用了不适合的激活函数（sigmoid）和预测方式（阈值判断）

**解决方案**：

将 `sigmoid` + 阈值判断改为 `softmax` + `argmax`：

1. **在 `evaluate` 方法中**（第48-58行）：
```python
# 修改前
_, _, logits = model(batch.x, batch.edge_index, batch.batch)
probs = torch.sigmoid(logits)
preds = (probs > 0.5).long()

# 修改后
_, _, logits = model(batch.x, batch.edge_index, batch.batch)
# 使用softmax获取概率分布
probs = torch.softmax(logits, dim=1)
# 取第二个类别（恶意类，索引1）的概率
probs_class1 = probs[:, 1]
# 使用argmax获取预测类别
preds = torch.argmax(probs, dim=1)
```

2. **在 `plot_confusion_matrix` 方法中**（第133-143行）：
```python
# 修改前
_, _, logits = model(batch.x, batch.edge_index, batch.batch)
probs = torch.sigmoid(logits)
preds = (probs > 0.5).long()

# 修改后
_, _, logits = model(batch.x, batch.edge_index, batch.batch)
# 使用softmax获取概率分布
probs = torch.softmax(logits, dim=1)
# 使用argmax获取预测类别
preds = torch.argmax(probs, dim=1)
```

3. **在 `plot_roc_curve` 方法中**（第189-199行）：
```python
# 修改前
_, _, logits = model(batch.x, batch.edge_index, batch.batch)
probs = torch.sigmoid(logits)

# 修改后
_, _, logits = model(batch.x, batch.edge_index, batch.batch)
# 使用softmax获取概率分布
probs = torch.softmax(logits, dim=1)
# 取第二个类别（恶意类，索引1）的概率用于ROC曲线
probs_class1 = probs[:, 1]
```

**关键改进**：
1. 使用 `softmax` 代替 `sigmoid`，确保概率归一化
2. 使用 `argmax` 获取预测类别，得到单标签格式
3. 对于ROC曲线，使用恶意类（索引1）的概率
4. 确保预测格式与标签格式一致

**技术要点**：
- **Sigmoid**：用于多标签分类，每个类别独立计算概率
- **Softmax**：用于单标签多分类，所有类别概率归一化，和为1
- 对于二分类任务，应该使用 `softmax` + `argmax`

**详细说明**：参见 [修复说明_二分类标签格式.md](./修复说明_二分类标签格式.md)

**状态**：✅ 已修复

## 技术总结

### 1. 类型注解最佳实践
- 使用类型注解时，确保从 `typing` 模块导入所需类型
- 常用类型：`Dict`, `List`, `Tuple`, `Optional`, `Union`, `Any`

### 2. 数据集划分注意事项
- 小数据集可能导致验证集或测试集为空
- 在使用验证集前检查其是否为空
- 考虑使用交叉验证处理小数据集

### 3. 训练-测试一致性原则
- **关键原则**：训练时和测试时的数据预处理必须完全一致
- 包括：
  - 特征维度
  - 特征归一化
  - 数据增强
  - 特征拼接顺序
- 违反此原则会导致模型无法正确预测

### 4. 层次化GNN架构
```
输入数据
    ↓
CFG-GNN (函数级)
    ↓ 128维嵌入
    ↓ 增强FCG特征
FCG-GNN (程序级)
    ↓ 输入256维 (128原始 + 128CFG嵌入)
    ↓ 输出128维嵌入
最终分类器
    ↓
预测结果
```
