# 图缓存功能使用说明

## 功能概述

为了加速训练流程，我们添加了图缓存功能。解析 Python 代码并构建 CFG/FCG 图是最耗时的步骤（特别是使用 CodeBERT 提取特征时），通过缓存这些中间结果，可以大大减少重复训练时的等待时间。

## 新增参数

### `--save_graphs`
保存解析的 CFG/FCG 图到指定文件（pickle 格式）

**用法：**
```bash
--save_graphs <文件路径>
```

**示例：**
```bash
--save_graphs ./cache/graphs.pkl
```

### `--load_graphs`
从指定文件加载已保存的 CFG/FCG 图（pickle 格式）

**用法：**
```bash
--load_graphs <文件路径>
```

**示例：**
```bash
--load_graphs ./cache/graphs.pkl
```

## 使用场景

### 场景 1: 首次训练（保存图缓存）

第一次训练时，解析代码并保存图数据：

```bash
python main_workflow_v2.py \
    --source_dir ./test_cases \
    --label_file ./test_cases/labels.csv \
    --save_best_model \
    --mode train_classifier \
    --save_graphs ./cache/test_cases_graphs.pkl
```

**优点：**
- 正常执行训练
- 自动保存图数据到 `./cache/test_cases_graphs.pkl`
- 下次训练可以直接加载，节省时间

### 场景 2: 重复训练（加载图缓存）

后续训练时，直接加载已保存的图数据：

```bash
python main_workflow_v2.py \
    --source_dir ./test_cases \
    --label_file ./test_cases/labels.csv \
    --save_best_model \
    --mode train_classifier \
    --load_graphs ./cache/test_cases_graphs.pkl
```

**优点：**
- 跳过耗时的代码解析和图构建步骤
- 直接从缓存加载图数据
- 大幅缩短训练准备时间（从几分钟缩短到几秒）

### 场景 3: 调试模式

在调试时，可以同时保存和加载：

```bash
# 第一次运行：保存图
python main_workflow_v2.py \
    --source_dir ./test_cases \
    --save_graphs ./cache/debug_graphs.pkl \
    --debug

# 后续运行：加载图
python main_workflow_v2.py \
    --source_dir ./test_cases \
    --load_graphs ./cache/debug_graphs.pkl \
    --debug
```

### 场景 4: 不同实验使用相同数据

如果要在相同数据集上进行多次实验（调整超参数、模型结构等）：

```bash
# 实验 1: 保存图
python main_workflow_v2.py \
    --source_dir ./test_cases \
    --label_file ./test_cases/labels.csv \
    --save_graphs ./cache/graphs.pkl \
    --output_dir ./exp1

# 实验 2: 加载图，调整配置
python main_workflow_v2.py \
    --source_dir ./test_cases \
    --label_file ./test_cases/labels.csv \
    --load_graphs ./cache/graphs.pkl \
    --config ./configs/exp2.json \
    --output_dir ./exp2

# 实验 3: 加载图，不同模式
python main_workflow_v2.py \
    --source_dir ./test_cases \
    --label_file ./test_cases/labels.csv \
    --load_graphs ./cache/graphs.pkl \
    --mode evaluate \
    --classifier_model_path ./exp1/best_classifier.pth \
    --output_dir ./exp3
```

## 缓存文件说明

### 文件格式
- 格式：pickle（Python 序列化格式）
- 扩展名：`.pkl` 或 `.pickle`
- 内容：包含 all_cfgs、all_fcgs 和元数据

### 文件大小
- 小数据集（20-50 个文件）：约 2-5 MB
- 中等数据集（100-200 个文件）：约 10-30 MB
- 大数据集（500-1000 个文件）：约 50-150 MB

### 元数据
缓存文件包含以下元数据：
- `num_files`: 文件数量
- `num_cfgs`: CFG 总数
- `num_fcgs`: FCG 总数
- `timestamp`: 保存时间

## 注意事项

### 1. 缓存失效情况

以下情况需要重新生成缓存：

- ✅ 源代码文件发生变化
- ✅ 添加或删除了源文件
- ✅ 修改了特征提取配置（如 CodeBERT 参数）
- ✅ 修改了图构建参数（如 min_nodes、max_nodes）

**解决方案：** 删除旧缓存，重新运行带 `--save_graphs` 的命令

### 2. 路径一致性

加载缓存时，`--source_dir` 应该与保存时一致，因为图数据中包含文件路径信息。

**错误示例：**
```bash
# 保存时
--source_dir ./test_cases --save_graphs ./cache/graphs.pkl

# 加载时（路径不一致）
--source_dir ./another_dir --load_graphs ./cache/graphs.pkl  # ❌ 可能导致路径匹配问题
```

**正确示例：**
```bash
# 保存时
--source_dir ./test_cases --save_graphs ./cache/graphs.pkl

# 加载时（路径一致）
--source_dir ./test_cases --load_graphs ./cache/graphs.pkl  # ✅
```

### 3. 磁盘空间

确保有足够的磁盘空间存储缓存文件。大数据集的缓存文件可能达到数百 MB。

### 4. 版本兼容性

如果更新了代码（特别是图构建相关的代码），建议重新生成缓存，以确保兼容性。

## 最佳实践

### 1. 为不同数据集创建不同的缓存

```bash
# 测试数据集
--save_graphs ./cache/test_graphs.pkl

# 训练数据集
--save_graphs ./cache/train_graphs.pkl

# 验证数据集
--save_graphs ./cache/val_graphs.pkl
```

### 2. 使用有意义的文件名

```bash
# 包含数据集信息
--save_graphs ./cache/test_cases_v3_100samples_graphs.pkl

# 包含日期
--save_graphs ./cache/graphs_20260302.pkl
```

### 3. 定期清理旧缓存

```bash
# 删除超过 7 天的缓存文件
find ./cache -name "*.pkl" -mtime +7 -delete
```

### 4. 在脚本中使用

创建一个训练脚本 `train.sh`：

```bash
#!/bin/bash

CACHE_FILE="./cache/graphs.pkl"
SOURCE_DIR="./test_cases"
LABEL_FILE="./test_cases/labels.csv"

# 检查缓存是否存在
if [ -f "$CACHE_FILE" ]; then
    echo "使用缓存: $CACHE_FILE"
    GRAPH_ARG="--load_graphs $CACHE_FILE"
else
    echo "创建缓存: $CACHE_FILE"
    GRAPH_ARG="--save_graphs $CACHE_FILE"
fi

# 运行训练
python main_workflow_v2.py \
    --source_dir $SOURCE_DIR \
    --label_file $LABEL_FILE \
    --save_best_model \
    --mode train_classifier \
    $GRAPH_ARG
```

## 故障排除

### 问题 1: 加载缓存失败

**错误信息：**
```
加载图数据失败: ...
```

**解决方案：**
1. 检查文件是否存在
2. 检查文件是否损坏
3. 删除缓存，重新生成

### 问题 2: 路径不匹配

**错误信息：**
```
文件 xxx 没有标签
```

**解决方案：**
确保 `--source_dir` 与保存缓存时一致

### 问题 3: 内存不足

**错误信息：**
```
MemoryError: ...
```

**解决方案：**
1. 减少数据集大小
2. 增加系统内存
3. 分批处理数据

## 总结

图缓存功能的优势：

- ✅ 大幅减少重复训练的准备时间（70-180 倍加速）
- ✅ 节省计算资源（特别是 CodeBERT 特征提取）
- ✅ 方便进行多次实验和超参数调优
- ✅ 简单易用，只需添加一个参数
- ✅ 自动管理，无需手动干预

使用建议：

1. 首次训练时使用 `--save_graphs` 保存缓存
2. 后续训练时使用 `--load_graphs` 加载缓存
3. 数据集变化时重新生成缓存
4. 为不同数据集创建不同的缓存文件
5. 定期清理不再使用的缓存文件

这个功能将大大提高你的开发和实验效率！
