# 3.4.3 层次化图神经网络模型

图神经网络（GNN）通过节点间的消息传递机制，能够有效捕获代码的结构依赖关系和行为模式，特别适合处理代码的非欧几里得结构数据。然而，现有GNN方法多倾向于单独处理某种类型的图或将不同粒度的图扁平化处理，导致信息丢失。CFG专注于函数内部的执行逻辑，FCG则描绘模块层面的函数调用关系，如何有效整合这两种层次的信息是提升代码表示质量的关键。

本文设计了层次化的图神经网络模型，通过两阶段训练策略实现从局部到全局的信息融合。第一阶段，CFG-GNN学习每个函数的精细化表示；第二阶段，将函数级CFG嵌入增强FCG节点特征，通过FCG-GNN学习程序整体嵌入。这种层次化方法不仅捕捉了函数内部的控制流语义，还通过信息传递机制使程序级表示能够感知其组成函数的深层语义，从而生成更具表达力的程序嵌入向量。模型整体框架如图3.X所示。

**CFG-GNN模型**采用图卷积网络（GCN）架构处理函数级控制流图。模型通过3层图卷积操作聚合邻居节点信息，每层图卷积后接ReLU激活函数和Dropout（0.5）增强表达能力和训练稳定性。图卷积层的计算公式为：

$$H^{(l+1)} = \sigma(\tilde{D}^{-\frac{1}{2}}\tilde{A}\tilde{D}^{-\frac{1}{2}}H^{(l)}W^{(l)})$$

其中，$\tilde{A} = A + I$为添加自环的邻接矩阵，$\tilde{D}$为度矩阵，$W^{(l)}$为可学习权重矩阵，$\sigma$为激活函数。最后通过全局平均池化操作将节点级嵌入聚合为图级嵌入：

$$h_{CFG} = \frac{1}{N}\sum_{i=1}^{N}h_i^{(L)}$$

生成64维的CFG结构特征向量。

**FCG-GNN模型**采用图注意力网络（GAT）架构处理程序级函数调用图。模型使用2层图注意力卷积以捕获复杂的调用关系，注意力机制对不同函数节点赋予不同权重，使模型能够关注更重要的函数。注意力系数计算公式为：

$$\alpha_{ij} = \frac{\exp(\text{LeakyReLU}(\mathbf{a}^T[Wh_i \| Wh_j]))}{\sum_{k \in \mathcal{N}(i)}\exp(\text{LeakyReLU}(\mathbf{a}^T[Wh_i \| Wh_k]))}$$

节点特征更新公式为：

$$h'_i = \sigma(\sum_{j \in \mathcal{N}(i)}\alpha_{ij}Wh_j)$$

通过注意力加权池化操作生成64维的FCG结构特征向量。

**特征融合策略**实现了CFG特征向FCG的传递。对于FCG中的每个函数节点，若该函数有对应的CFG，则使用训练好的CFG-GNN模型生成该函数的CFG嵌入，并将其拼接到FCG节点的原始特征上：

$$x'_i = [x_i \| h_{CFG}(f_i)]$$

若函数没有CFG（如外部函数），则使用零向量填充。这样，FCG节点特征既包含了函数自身的属性，又包含了函数内部的控制流结构信息，实现了函数级和程序级特征的有机融合。

层次化GNN模型的训练过程如算法3.X所示。训练分为两个阶段：第一阶段使用边预测任务训练CFG-GNN，通过区分真实边和负采样边学习函数内部控制流模式；第二阶段利用训练好的CFG-GNN为FCG节点生成增强特征，然后训练FCG-GNN学习程序级表示。损失函数结合边预测损失和分类损失：

$$\mathcal{L} = \mathcal{L}_{edge} + \lambda \mathcal{L}_{cls}$$

其中，$\mathcal{L}_{edge}$为二元交叉熵损失，$\mathcal{L}_{cls}$为交叉熵分类损失，$\lambda$为平衡系数。

**算法3.X：层次化图神经网络训练**

**输入**：CFG数据集 $\mathcal{D}_{CFG}$，FCG数据集 $\mathcal{D}_{FCG}$  
**输出**：训练好的CFG-GNN和FCG-GNN模型

1. 初始化CFG-GNN和FCG-GNN模型参数
2. **// 阶段一：训练CFG-GNN**
3. **for** epoch = 1 **to** $E_{cfg}$ **do**
4. &nbsp;&nbsp;&nbsp;&nbsp;**for** batch **in** $\mathcal{D}_{CFG}$ **do**
5. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;$h_{node}, h_{graph} \leftarrow$ CFG-GNN(batch.x, batch.edge_index)
6. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;生成负样本边 $E_{neg}$
7. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;$\mathcal{L}_{edge} \leftarrow$ BCE(边预测, 真实边)
8. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;更新CFG-GNN参数
9. &nbsp;&nbsp;&nbsp;&nbsp;**end for**
10. **end for**
11. **// 阶段二：特征增强与FCG-GNN训练**
12. **for** fcg **in** $\mathcal{D}_{FCG}$ **do**
13. &nbsp;&nbsp;&nbsp;&nbsp;**for** $node_i$ **in** fcg.nodes **do**
14. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;$h_{cfg} \leftarrow$ CFG-GNN(CFG($f_i$))
15. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;$x'_i \leftarrow$ concat($x_i$, $h_{cfg}$)
16. &nbsp;&nbsp;&nbsp;&nbsp;**end for**
17. **end for**
18. **for** epoch = 1 **to** $E_{fcg}$ **do**
19. &nbsp;&nbsp;&nbsp;&nbsp;**for** batch **in** 增强的$\mathcal{D}_{FCG}$ **do**
20. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;$h_{node}, h_{graph} \leftarrow$ FCG-GNN(batch.x', batch.edge_index)
21. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;$\mathcal{L} \leftarrow \mathcal{L}_{edge} + \lambda \mathcal{L}_{cls}$
22. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;更新FCG-GNN参数
23. &nbsp;&nbsp;&nbsp;&nbsp;**end for**
24. **end for**
25. **return** CFG-GNN, FCG-GNN

---

模型实现基于PyTorch Geometric框架，CFG-GNN配置为3层GCN（隐藏维度32，输出维度64，Dropout率0.5），FCG-GNN配置为2层GAT（隐藏维度32，输出维度64，Dropout率0.5）。优化器采用Adam（学习率0.001，权重衰减5e-4），CFG-GNN和FCG-GNN分别训练50轮。通过这种层次化设计，模型生成的程序嵌入向量能够同时感知函数内部的控制流语义和程序整体的调用结构，为后续恶意代码检测任务提供了强有力的特征支持。
