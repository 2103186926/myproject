# 二分类标签格式不匹配问题修复说明

## 问题描述

在模型评估阶段出现以下错误：

```
ValueError: Classification metrics can't handle a mix of binary and multilabel-indicator targets
```

## 问题原因

### 根本原因
模型输出的预测格式与真实标签格式不一致：

1. **模型输出**：
   - `logits` 形状：`[batch_size, 2]`（二分类的两个类别）
   - 使用 `torch.sigmoid(logits)` 后：`[batch_size, 2]`
   - 使用 `(probs > 0.5).long()` 后：`[batch_size, 2]` 的多标签格式

2. **真实标签**：
   - `batch.y` 形状：`[batch_size]`（单标签格式）
   - 值为 0（正常）或 1（恶意）

3. **sklearn期望**：
   - 对于二分类任务，预测和标签都应该是一维数组
   - 预测值：`[0, 1, 0, 1, ...]`
   - 标签值：`[0, 1, 0, 1, ...]`

### 错误传播路径
```
model.forward() 
  → logits [batch_size, 2]
  → torch.sigmoid(logits) [batch_size, 2]
  → (probs > 0.5).long() [batch_size, 2]  # 多标签格式
  → confusion_matrix(labels, preds)  # labels是[batch_size]，preds是[batch_size, 2]
  → ValueError: 格式不匹配
```

## 解决方案

### 核心修改
将 `sigmoid` + 阈值判断改为 `softmax` + `argmax`：

**修改前**（错误的方式）：
```python
_, _, logits = model(batch.x, batch.edge_index, batch.batch)
probs = torch.sigmoid(logits)  # [batch_size, 2]
preds = (probs > 0.5).long()    # [batch_size, 2] 多标签格式
```

**修改后**（正确的方式）：
```python
_, _, logits = model(batch.x, batch.edge_index, batch.batch)
# 使用softmax获取概率分布
probs = torch.softmax(logits, dim=1)  # [batch_size, 2]
# 取第二个类别（恶意类，索引1）的概率
probs_class1 = probs[:, 1]  # [batch_size]
# 使用argmax获取预测类别
preds = torch.argmax(probs, dim=1)  # [batch_size] 单标签格式
```

## 技术要点

### 1. Sigmoid vs Softmax

**Sigmoid**（用于多标签分类）：
- 每个类别独立计算概率
- 输出范围：[0, 1]
- 各类别概率之和可以 > 1
- 适用场景：一个样本可以属于多个类别

**Softmax**（用于单标签多分类）：
- 所有类别概率归一化
- 输出范围：[0, 1]
- 各类别概率之和 = 1
- 适用场景：一个样本只属于一个类别

### 2. 二分类的正确处理方式

对于二分类任务（正常 vs 恶意）：

```python
# 模型输出
logits = model(...)  # [batch_size, 2]

# 方式1：使用softmax + argmax（推荐）
probs = torch.softmax(logits, dim=1)  # [batch_size, 2]
preds = torch.argmax(probs, dim=1)    # [batch_size]

# 方式2：使用softmax + 阈值
probs = torch.softmax(logits, dim=1)  # [batch_size, 2]
preds = (probs[:, 1] > 0.5).long()    # [batch_size]

# 方式3：只输出一个logit（需要修改模型）
logits = model(...)  # [batch_size, 1]
probs = torch.sigmoid(logits)         # [batch_size, 1]
preds = (probs > 0.5).long()          # [batch_size, 1]
```

### 3. ROC曲线的概率选择

对于二分类的ROC曲线：
- 需要使用**正类（恶意类）的概率**
- 在我们的场景中，恶意类是索引1
- 因此使用 `probs[:, 1]`

```python
probs = torch.softmax(logits, dim=1)  # [batch_size, 2]
probs_class1 = probs[:, 1]  # [batch_size] 恶意类的概率
# 用于计算ROC曲线
fpr, tpr, _ = roc_curve(all_labels, probs_class1)
```

## 相关问题

### Q1: 为什么之前使用sigmoid？
**A**: 可能是从多标签分类的代码模板复制过来的，没有针对二分类场景进行调整。

### Q2: 如果模型只输出一个logit怎么办？
**A**: 如果模型输出 `[batch_size, 1]`，则应该使用sigmoid：
```python
logits = model(...)  # [batch_size, 1]
probs = torch.sigmoid(logits).squeeze()  # [batch_size]
preds = (probs > 0.5).long()  # [batch_size]
```

### Q3: 损失函数需要修改吗？
**A**: 不需要。`nn.BCEWithLogitsLoss()` 可以处理 `[batch_size, 2]` 的logits，它会自动处理。但如果想更明确，可以使用 `nn.CrossEntropyLoss()`。

## 后续优化建议

### 1. 统一损失函数
使用 `nn.CrossEntropyLoss()` 更适合单标签多分类：
```python
criterion = nn.CrossEntropyLoss()
loss = criterion(logits, labels)  # logits: [batch_size, 2], labels: [batch_size]
```

### 2. 添加维度检查
在评估前检查维度：
```python
def evaluate(self, model, test_loader, task='binary'):
    # ...
    if task == 'binary':
        _, _, logits = model(...)
        assert logits.dim() == 2 and logits.size(1) == 2, \
            f"二分类任务期望logits形状为[batch_size, 2]，实际为{logits.shape}"
```

### 3. 封装预测逻辑
创建统一的预测函数：
```python
def predict_binary(logits):
    """二分类预测"""
    probs = torch.softmax(logits, dim=1)
    preds = torch.argmax(probs, dim=1)
    probs_class1 = probs[:, 1]
    return preds, probs_class1
```

## 总结

这个问题是典型的**分类任务类型混淆**问题：
- 错误地将**单标签二分类**当作**多标签分类**处理
- 使用了不适合的激活函数（sigmoid）和预测方式（阈值判断）
- 导致预测格式与标签格式不匹配

通过改用 `softmax` + `argmax`，确保了预测格式与标签格式一致，问题得以解决。
