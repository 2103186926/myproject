# 测试用例生成器使用说明

## 概述

`test_case_generator.py` 是一个用于生成海洋科学云计算平台测试用例的工具，可以自动生成指定数量的正常代码和恶意代码样本。

## 功能特点

1. **自动生成测试用例**：支持生成任意数量的正常和恶意代码
2. **真实业务场景**：所有代码都基于真实的海洋科学计算任务
3. **复杂的控制流**：包含循环、条件、异常处理等复杂结构
4. **丰富的函数调用**：体现CFG和FCG的复杂关系
5. **变体生成**：基于模板自动生成代码变体
6. **自动标签**：自动生成labels.csv标签文件

## 使用方法

### 基本用法

```bash
# 使用默认参数（8个正常 + 7个恶意）
python test_case_generator.py

# 指定输出目录
python test_case_generator.py --output_dir ./my_test_cases

# 自定义数量
python test_case_generator.py --num_normal 10 --num_malicious 10

# 指定总数（自动平均分配）
python test_case_generator.py --total 20
```

### 命令行参数

- `--output_dir`: 输出目录（默认: `./test_cases`）
- `--num_normal`: 正常代码数量（默认: 8）
- `--num_malicious`: 恶意代码数量（默认: 7）
- `--total`: 总数量（如果指定，将平均分配正常和恶意代码）

## 生成策略

### 正常代码

**基础模板**（2个）：
1. **CTD数据处理系统**：复杂的数据处理流程，包含质量控制、插值、平滑等
2. **模式后处理系统**：海洋数值模式输出分析，包含流场分析、涡旋识别等

**变体生成**：
- 在文档字符串中添加变体标记
- 修改部分数值参数（前3个浮点数常量）
- 保持代码结构和逻辑不变

### 恶意代码

**基础模板**（3个）：
1. **文件系统逃逸**：尝试访问宿主机文件系统、挂载、读取敏感文件
2. **权限提升**：尝试修改权限、设置SUID、修改cgroup限制
3. **Namespace逃逸**：尝试unshare系统调用、修改cgroup、写入SSH密钥

**变体生成**：
- 在文档字符串中添加恶意变体标记
- 添加额外的恶意操作（whoami、id等）
- 保持原有恶意行为不变

## 代码特点

### CFG（控制流图）特点

- **条件分支**：多层if-else嵌套
- **循环结构**：for循环、while循环
- **异常处理**：try-except-finally块
- **复杂控制流**：多个return点、break/continue

### FCG（函数调用图）特点

- **main函数入口**：所有程序都以main()为入口
- **类方法调用**：面向对象编程，方法间相互调用
- **独立函数**：工具函数和计算函数
- **多层调用**：函数调用深度达到3-4层

## 与主工作流集成

生成测试用例后，可以直接用于主工作流：

```bash
# 标准训练
python main_workflow_v2.py \
    --source_dir ./test_cases \
    --label_file ./test_cases/labels.csv \
    --output_dir ./output

# K折交叉验证
python main_workflow_v2.py \
    --source_dir ./test_cases \
    --label_file ./test_cases/labels.csv \
    --output_dir ./output_kfold \
    --use_kfold \
    --n_folds 5
```

## 标签文件格式

`labels.csv` 文件格式：

```csv
file_path,label
结构特征提取/test_cases/normal/ctd_data_processor.py,0
结构特征提取/test_cases/normal/model_postprocessing.py,0
结构特征提取/test_cases/malicious/malicious_ocean_processor_1.py,1
...
```

- **label=0**：正常代码
- **label=1**：恶意代码

## 扩展建议

如果需要更多样化的测试用例，可以：

1. **添加新的基础模板**：在`_generate_normal_cases()`和`_generate_malicious_cases()`中添加新的代码模板
2. **改进变体生成**：在`_create_normal_variant()`和`_create_malicious_variant()`中实现更复杂的变体逻辑
3. **参数化生成**：使用模板引擎（如Jinja2）实现更灵活的代码生成

## 注意事项

1. **不要在生产环境运行恶意代码**：这些代码包含真实的攻击逻辑
2. **使用容器隔离**：建议在Docker容器中测试
3. **监控系统资源**：注意CPU、内存、网络使用情况
4. **定期更新**：根据新的攻击手法更新恶意代码模板
